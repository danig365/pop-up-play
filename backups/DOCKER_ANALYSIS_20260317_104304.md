## Backup Strategy

### Database Backups
1. **SQL Format** (.sql files)
   - Human-readable SQL statements
   - Good for version control and review
   - Larger file size

2. **Custom Format** (.dump files)
   - Binary PostgreSQL custom format
   - More efficient for restoration
   - Supports parallel restore
   - Better for large databases

### Code Backups
1. **Source Code Archive** (.tar.gz)
   - Excludes:
     - node_modules (can be reinstalled)
     - .git (version control)
     - dist/ (build artifacts)
     - .env/.env.local (sensitive)
     - uploads/ (user data)
   - Includes all source code and configuration

2. **Docker Configuration**
   - Container inspection data (JSON)
   - Volume configuration
   - Network setup

3. **Database Structure**
   - schema.sql backup
   - seed.sql backup
   - Migration files

## Restoration Procedures

### Database Restoration

**From SQL backup:**
```bash
docker exec -i pop-up-play-db psql -U popupapp popup_play_db < backup.sql
```

**From custom format backup:**
```bash
docker exec -i pop-up-play-db pg_restore -U popupapp -d popup_play_db backup.dump
```

### Code Restoration

```bash
cd /path/to/restore
tar -xzf popup_play_code_*.tar.gz
npm install
docker-compose up -d
```
